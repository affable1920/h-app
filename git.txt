// What to commit and not to commit to git (.gitignore)

What TO Commit:
✅ Source code - .js, .py, .java, .css, etc.
✅ Configuration templates - .env.example (without real secrets)
✅ Documentation - README.md, docs/
✅ Package definitions - package.json, requirements.txt, Cargo.toml
✅ Tests - test files and test data
✅ Assets - images, fonts (if not too large)

What NOT TO Commit:
❌ Dependencies - others can install them via package.json
❌ Secrets - API keys, passwords, tokens
❌ Build artifacts - compiled code, minified files
❌ Large files - videos, large datasets (use Git LFS if needed)
❌ Local config - database URLs, local paths
❌ IDE settings - personal editor configurations
